## 第1套：基础文件操作（难度：★）

**题目：批量文件重命名**

### 第1轮：0-1代码生成

```
请编写一个bat脚本，实现以下功能：
- 将当前目录下所有 `.txt` 文件重命名，在原文件名前添加当前日期（格式：YYYYMMDD）
- 例如：`report.txt` → `20260424_report.txt`
- 显示重命名前后对比列表
- 执行完成后给出统计信息（处理了多少个文件）
```

### 第2轮：Bug修复

```
执行你生成的脚本时出现以下问题：
- 当文件名中包含空格时（如 "my report.txt"），脚本无法正确处理，报错"系统找不到文件"
- 请定位问题原因并修复脚本
```

### 第3轮：Feature迭代

```
请在现有脚本基础上增加以下功能：
- 支持用户指定要处理的文件扩展名（不只是.txt，可以是任意扩展名）
- 支持用户选择日期格式：YYYYMMDD 或 YYYY-MM-DD
- 添加命令行参数解析功能，如：rename.bat -ext log -format YYYY-MM-DD
```

### 第4轮：代码理解

```
请解释你脚本中以下代码片段的工作原理：
- `%date%` 变量的处理逻辑
- `for` 循环中的延迟变量扩展 `!var!` 与 `%var%` 的区别
- 为什么需要 `setlocal enabledelayedexpansion`
```

### 第5轮：工程化

```
请将脚本工程化改进：
- 添加详细的帮助文档（--help 参数）
- 添加版本信息和作者信息
- 创建日志文件记录每次执行的操作
- 添加错误码定义，便于脚本集成到其他自动化流程中
```

| 第5轮 | 工程化 | 测试工程实践和系统集成能力 |

### 评分维度

1. **代码正确性**：生成的代码是否能正确运行
2. **代码质量**：结构清晰、命名规范、可维护性
3. **问题解决能力**：Bug定位准确性和修复效率
4. **功能扩展能力**：新功能实现是否合理、是否与原有代码兼容
5. **工程实践**：日志、错误处理、文档、配置管理等
